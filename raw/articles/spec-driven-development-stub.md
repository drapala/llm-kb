## Claims descritos pelo usuário

- **Spec antes de código:** desenvolvimento deve começar com especificação formal, não com código
- **Codebase como especificação de facto é anti-pattern:** quando o código se torna a única "especificação" do sistema, o entendimento se degrada e agentes não têm âncora normativa para decisões de design

## Por que isso importa

A codebase como spec de facto é o estado natural de projetos que crescem sem documentação — especialmente com agentes AI que acumulam código sem manter spec correspondente. Este anti-pattern é análogo a "configuration debt" (Sculley 2015) — onde configurações se tornam a única fonte de verdade porque nunca foram documentadas formalmente.
