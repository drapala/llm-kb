We consider an adaptive black-box adversary who can create arbitrary accounts, issue strategically timed queries, and run probes concurrently or in batches. They can coordinate across accounts, replay prompts with controlled prefixes, and adaptively refine probes to test whether a specific prefix or span has been cached by another tenant. This unified hierarchy maintains both scalability and privacy integrity. Each node stores two metadata fields:\nprivate_tag\n(0 for shareable, 1 for private) and\ncreator_id\n. This suggests treating detection as a pluggable policy module while keeping cache enforcement unchanged.\n9.\nRelated Work\n9.1.\nMulti-tenant Security\nSide-channel attacks have long threatened multi-tenant systems due to shared resource usage. They can be broadly categorized into classical system-level attacks and emerging model-level side channels.\nClassical side-channel attacks exploit shared hardware or OS abstractions in multi-tenant systems. Cross-VM attacks recover sensitive data between co-located virtual machines by monitoring CPU caches or memory access patterns\n(Zhang\net al.\n,\n2012\n,\n2014\n; Varadarajan\net al.\n,\n2015\n; Xiao\net al.\n,\n2016\n)\n.\nOther works leverage shared OS resources, such as OS data structure or public file systems, to launch cross-application attacks in Unix, Android, or iOS environments\n(Zhang and Wang,\n2009\n; Jana and Shmatikov,\n2012\n; Chen\net al.\n,\n2014\n; Zhang\net al.\n,\n2016\n,\n2018\n; Wang\net al.\n,\n2023\n)\n. Recent studies further reveal that multiple WebAssembly modules isolated in the same runtime are vulnerable to cross-module attacks\n(Narayan\net al.\n,\n2021\n; Kocher\net al.\n,\n2019\n)\n.\nComplementing these, recent work highlights a new class of attacks targeting LLM inference backends. These attacks exploit timing differences caused by shared KV-cache reuse to infer user inputs\n(Wu\net al.\n,\n2025\n; Zheng\net al.\n,\n2024b\n; Song\net al.\n,\n2024\n)\n. For example, PromptPeek\n(Wu\net al.\n,\n2025\n)\nand InputSnatch\n(Zheng\net al.\n,\n2024b\n)\nreconstruct user prompts via TTFT measurements in black-box settings. Unlike classical channels, these attacks are unique to LLM-serving pipelines, where performance optimizations inadvertently expose privacy risks. Our work builds on these findings by proposing a practical, multi-tier defense system that addresses this emerging attack surface.\n9.2.\nDefenses Against KV-cache Side Channels\nTo mitigate KV-cache side-channel attacks, researchers and practitioners have proposed a range of defenses.\nThe most straightforward solution is\nuser-level cache isolation\n: it prevents cross-user sharing by allocating distinct cache namespaces per user. This containment strategy eliminates cache-based interactions between users and is adopted by some LLM providers (OpenAI\n(OpenAI,\n2024b\n)\nand DeepSeek\n(DeepSeek,\n2024\n)\n) and by researchers\n(Pang\net al.\n,\n2024\n)\nfor prefix caching. While effective, such strict isolation sacrifices memory efficiency and undermines the performance benefits of cache reuse.\nRate Limiting\nserves as a complementary defense by throttling the frequency of user queries, thereby impeding rapid probing required for cache-timing attacks. For instance, OpenAI enforces rate limits to prevent abuse and ensure infrastructure stability\n(OpenAI,\n2024a\n)\n. However, rate limiting must be carefully tuned to avoid degrading service quality for benign users.\nA third line of defense is\nTiming Obfuscation\n, which aims to eliminate observable latency differences between cache hits and misses. Prior work in model extraction has shown that response time can correlate with internal architecture details\n(Batina\net al.\n,\n2019\n; Duddu\net al.\n,\n2018\n; Dong\net al.\n,\n2019\n)\n, motivating similar countermeasures in LLM serving. Two common strategies are: (i) enforcing constant-time execution\n(Maji\net al.\n,\n2021\n)\nor injecting random delays to flatten latency variance\n(Breier\net al.\n,\n2023\n)\n, and (ii) disabling token-level streaming, which removes fine-grained timing signals from observable outputs. While these approaches can mask timing patterns, they often incur latency penalties or reduce the interactivity of real-time systems.\n10.\nConclusion\nThis paper presented SafeKV, a system-level framework for privacy-preserving KV-cache management in LLM serving. The design integrates hybrid privacy detection with cache-level sensitivity control to mitigate timing side channels introduced by shared caches. By combining rule-based filtering, lightweight model classification, and context-aware validation with a unified radix-tree cache manager, SafeKV enables selective reuse of non-sensitive entries while securely isolating private content. Evaluations across multiple LLMs show that SafeKV improves throughput by up to\n2.66\n\u00d7\n2.66\\times\ncompared to per-user cache isolation. Our experiments confirm that privacy-aware cache sharing is not only feasible but essential for scalable and secure LLM serving.\nReferences\n[1]\n(2024)\nAb-ai/pii_model\n.\nNote:\nhttps://huggingface.co/ab-ai/pii_model\nCited by:\nTable 6\n.\n[2]\n(2024)\nAi4privacy/pii-masking-200k\n.\nNote:\nhttps://huggingface.co/datasets/ai4privacy/pii-masking-200k\nCited by:\n\u00a77.4\n,\n\u00a77.4\n.\n[3]\n(2023)\nAnon8231489123/sharegpt_vicuna_unfiltered\n.\nNote:\nhttps://huggingface.co/datasets/anon8231489123/ShareGPT_Vicuna_unfiltered\nCited by:\nTable 4\n,\n\u00a77.4\n,\n\u00a77.4\n.\nAnthropic (2024)\nPrompt caching with claude.\n.\nNote:\nAccessed: 2025-07-17\nExternal Links:\nLink\nCited by:\nTable 2\n.\nF.
